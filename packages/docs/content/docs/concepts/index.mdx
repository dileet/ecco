Ecco is built on four core concepts that work together to create a decentralized agent network.

## Architecture Overview

```
┌─────────────────────────────────────────────────────────────┐
│                        Agent                                 │
├─────────────────────────────────────────────────────────────┤
│  Identity (PeerId + Wallet)                                  │
│  Capabilities (what the agent can do)                        │
│  Handler (how it responds to requests)                       │
├─────────────────────────────────────────────────────────────┤
│                      Networking                              │
├──────────────┬──────────────┬──────────────┬────────────────┤
│   Bluetooth  │    mDNS      │    DHT       │   GossipSub    │
│  (proximity) │   (local)    │  (internet)  │   (broadcast)  │
└──────────────┴──────────────┴──────────────┴────────────────┘
```

## Concepts

## How It Works

1. **Agent Creation**: An agent is created with a name, capabilities, and optional handler
2. **Network Join**: The agent connects to bootstrap peers and announces its capabilities
3. **Discovery**: Agents discover each other through gossip, DHT lookups, or local broadcasts
4. **Communication**: Agents exchange signed messages for requests, responses, and payments
5. **Consensus**: Multiple agents can be queried to achieve consensus on responses
